# xgbp_0.1

Standalone Compact XGBP application.

## Contents

```text
contract/src/compact/examples/XGBP.compact        # contract entrypoint
contract/src/compact/token/                      # CFT source
contract/src/artifacts/XGBP/                     # generated by Compact Tools
contract/dist/artifacts/XGBP/                    # packaged build output
cli/src/main.ts                                  # minimal CLI
cli/src/tui.ts                                  # fixed-screen terminal UI
infra/standalone.yml                            # local Midnight network
```

Generated artifacts are ignored by git.

## Requirements

- Node.js 20+
- Docker with `docker compose`
- Midnight Compact toolchain on `PATH`

Check the Compact toolchain:

```bash
compact compile --version
```

## Fresh Terminal

```bash
cd /home/g/June_Context/xgbp_0.1
npm ci
npm run build
npm run typecheck
npm run cli -- networks
```

`npm run build` uses OpenZeppelin Compact Tools:

```bash
compact-builder +0.31.0 --clean-dist --src src --dir compact/examples --out src/artifacts
```

For compile-only work:

```bash
npm run compact
```

Normal CLI commands do not rebuild. Run `npm run build` again only after
changing contract source or TypeScript.

## Local Network

If `network:up` reports ports in use, stop the other local Midnight stack first.

## Deploy

Deploy to the local network:

```bash
npm run deploy -- --network local --name XGBP --symbol XGBP --decimals 2 --vk-chunk-size 8
```

If you want deploy to force a fresh Compact/TypeScript build first:

```bash
npm run deploy:fresh -- --network local
```

Local deploy defaults to the standalone genesis wallet seed.

Use a fixed wallet seed for repeatable testing:

```bash
npm run deploy -- --network local --seed <hex-seed>
```

The CFT verifier keys are published after the stripped deploy. `--vk-chunk-size`
controls how many keys are attempted per maintenance transaction; if a chunk is
too large for standalone block limits, the CLI halves the chunk and retries.

Deployment records are written under `.xgbp/deployments/` and are ignored by git.

## Fixed-Screen TUI

Start from a fresh terminal:

```bash
cd /home/g/June_Context/xgbp_0.1
npm run network:up
npm run tui -- --network local
```

The TUI requires a terminal of at least `120x35`.

Current startup options:

```text
1 Deploy new XGBP contract
2 Exit
```

After deploy, the dashboard shows Issuer, Alice, Bob, the public contract view,
the KYC/freeze registry, and a fixed activity panel. Values are updated only
after real contract calls succeed; non-owner balances are shown as `encrypted`.

## Current Boundary

OpenZeppelin Compact Tools are used for Compact compilation and contract-package build assembly. Deployment uses the Midnight JS SDK because Compact Tools do not provide network submission.
